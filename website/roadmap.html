Roadmap

From Alpha to Production

Ryx follows a structured, phase-based development plan. Each phase unlocks new capabilities while maintaining our core guarantees of safety and performance.

📍 Currently: Phase 1 — Alpha Track on GitHub
Overall Progress Phase 1 / 5
Spec & Design Language Complete Ecosystem Stable 1.0
Active

Phase 1 — Language Foundation

01

Establishing the core language specification, type system, module system, and the DASO memory model in the compiler frontend.

✅ Language specification (20 chapters)
✅ DASO memory model designed
✅ Keyword set finalized (~25)
✅ Spine_Case naming convention
✅ Module system designed
🔄 Compiler frontend (Rust)
🔄 Ryx → C transpilation
⬜ Basic standard library
⬜ TCC integration (ryx run)
Upcoming

Phase 2 — Core Compiler

02

Complete the Ryx-to-C compiler with full DASO region management, escape analysis, and type inference.

⬜ Full DASO region compiler
⬜ Escape analysis engine
⬜ Ownership checker
⬜ Type inference system
⬜ GCC / Clang backend integration
⬜ Error messages (human-friendly)
⬜ Debug info / symbols
Upcoming

Phase 3 — Standard Library

03

Build a comprehensive, high-quality standard library with all essential modules for real-world development.

⬜ std.io — I/O operations
⬜ std.net — networking
⬜ std.fs — filesystem
⬜ std.json — JSON parsing
⬜ std.math — math functions
⬜ std.crypto — cryptography
⬜ std.tensor — AI/ML ops
⬜ std.async — async runtime
Upcoming

Phase 4 — Tooling & Ecosystem

04

Developer tooling, package manager, IDE support, and ecosystem infrastructure for the Ryx community.

⬜ Package manager (ryx pkg)
⬜ Language server (LSP)
⬜ VS Code extension
⬜ Formatter (ryx fmt)
⬜ Linter (ryx lint)
⬜ Test runner (ryx test)
⬜ Package registry
⬜ C FFI bindings
Horizon

Phase 5 — Ryx 1.0 Stable

Production-ready release. Battle-tested compiler, comprehensive stdlib, thriving package ecosystem, excellent documentation, and stability guarantees. The moment Ryx becomes a dependable foundation for real projects.

05
Stable ABI Self-hosting 1000+ packages Multi-platform Production tested Security audited
