Getting Started

Welcome to Ryx

Ryx is a modern, statically typed systems programming language designed for developers who want C-level performance, memory safety without a garbage collector, and a clean, expressive syntax. You write simple code — the compiler does the complex reasoning.

Design goal: "High performance, strong safety, deterministic behavior, and low cognitive load are not trade-offs. They are design choices."

Ryx is suitable for: compilers, game engines, databases, high-performance servers, AI/ML infrastructure, embedded systems, and operating system components.

Hello, Ryx!

Create a file hello.ryx:

hello.ryx
use std.io

act main() {
    std.io.println("Hello, Ryx!")
}

Run it with:

ryx run hello.ryx

For a production binary:

ryx build hello.ryx

The Toolchain

Ryx has two build modes:

  • ryx run — Uses TCC (Tiny C Compiler) for near-instant compilation. Fast iteration loop during development.
  • ryx build — Targets GCC or Clang for fully optimized production binaries.

The Ryx compiler is a frontend written in Rust that transpiles your .ryx source to C11–C23, which is then compiled by the chosen backend.

Variables

Variables in Ryx are immutable by default. Use := for immutable (type inferred) and mut for mutable.

variables.ryx
// Immutable — type inferred
name  := "Ryx"
count := 42
pi    := 3.14159

// Explicit type (Constant Syntax: Name Space Type)
user_id int   := 1
weight  float := 72.5

// Mutable variable
mut score int = 0
score = 99   // allowed — score is mutable

// Compile-time constant (ALL_CAPS convention)
const MAX_SIZE int = 1024
const VERSION  str = "1.0.0"

// Reference-counted shared value (opt-in)
shared cache := Map[str, int]()

Naming Convention: Spine_Case

Ryx uses Spine_Case — underscores between words, capitalize the most important word.

naming.ryx
user_Account        // Account is the key concept
get_User_By_Id      // User is what we're getting
process_Payment     // Payment is the focus
email_Address       // Address is the property
MAX_CONNECTIONS     // Constant — ALL_CAPS

Built-in Types

All types are available without imports. Optimized for 64-bit systems.

types.ryx
// Primitives
a bool   := true
b int    := 42          // signed 64-bit
c uint   := 100u        // unsigned 64-bit
d float  := 3.14        // 64-bit float
e str    := "hello"     // UTF-8 string
f bytes  := bytes[1024] // raw byte array
g void                  // no return value

// AI/ML primitive — first-class tensor type
weights tensor := tensor[[0.1, 0.2, 0.3]]

// Standard library types (always available)
opt  Option[int]       := Some(42)
res  Result[int, str]  := Ok(100)

// Hex, binary, octal integer literals
hex_val    := 0xFF
binary_val := 0b1010
octal_val  := 0o755

// String interpolation
greeting := "Hello, {e}! The answer is {b}."

Functions

Functions are defined with the act keyword. They can use expression body syntax with -> for one-liners.

functions.ryx
// One-liner with expression body
act square(n int) int -> n * n
act add(a int, b int) int -> a + b

// Full body with retn
act greet(name str) str {
    msg := "Hello, {name}!"
    retn msg
}

// Generic function
act first[T](arr [T]) Option[T] {
    if arr.len() == 0 { retn None }
    retn Some(arr[0])
}

// Public function
pub act calculate(x float, y float) float -> x * x + y * y

// guard — flat early exit (replaces nested ifs)
act process(user User) Result[str, str] {
    guard user.name != "" else { retn Err("Name required") }
    guard user.id > 0    else { retn Err("Invalid ID") }
    retn Ok("Processed: {user.name}")
}

Control Flow

Ryx uses if / else, loop, match, and guard. No parentheses required in conditions.

control.ryx
// if / else — no parentheses needed
if score > 90 {
    println("Excellent!")
} else if score > 70 {
    println("Good")
} else {
    println("Keep going")
}

// loop — universal loop (replaces for/while)
loop i in 0..10 {
    println(i)
}

// loop with collection
loop item in names {
    println(item)
}

// infinite loop with break
mut count int = 0
loop {
    count += 1
    if count == 10 { break }
}

// match — powerful pattern matching
match status {
    Status.Active       -> println("Online"),
    Status.Idle         -> println("Away"),
    Status.Offline(msg) -> println("Gone: {msg}"),
}

// match with value
result := match score {
    90..=100 -> "A",
    70..=89  -> "B",
    50..=69  -> "C",
    _        -> "F",
}

// membership check
if "Alice" in admin_list {
    println("Admin access granted")
}

Structs & Enums

structs.ryx
// Struct definition
struct User {
    id    int,
    name  str,
    email str,
    score float,
}

// Create instance — Constant Syntax (no = needed)
user := User {
    id    1,
    name  "Prathmesh",
    email "p@ryx.dev",
    score 99.5,
}

// Access with dot operator
println(user.name)       // "Prathmesh"
println(user.score)      // 99.5

// Enum — named variants
enum Color {
    Red,
    Green,
    Blue,
    Custom(int, int, int),   // variant with data
}

// Enum usage
bg_color := Color.Blue
fg_color := Color.Custom(255, 128, 0)

// Type alias
alias UserMap  = Map[int, User]
alias Vector3  = [float; 3]
alias Callback = act(str) void

Traits & Impl

Traits define shared behaviors. Use impl to implement them for a type.

traits.ryx
// Define a trait
trait Animal {
    act name(self) str
    act sound(self) str
    act describe(self) str  // default impl
}

struct Dog {
    breed str,
}

struct Cat {
    indoor bool,
}

// Implement trait for Dog
impl Animal for Dog {
    act name(self) str -> "Dog"
    act sound(self) str -> "Woof!"
    act describe(self) str ->
        "{self.name()} ({self.breed}) says {self.sound()}"
}

// Implement trait for Cat
impl Animal for Cat {
    act name(self) str -> "Cat"
    act sound(self) str -> "Meow!"
    act describe(self) str ->
        "{self.name()} says {self.sound()}"
}

// Polymorphic usage
d := Dog { breed "Labrador" }
c := Cat { indoor true }

println(d.describe())
println(c.describe())

DASO — Memory Model

DASO (Deterministic Automatic Scoped Ownership) is Ryx's memory model. You almost never think about memory — the compiler handles it.

The mental model: "If it doesn't escape, I don't think about memory. If it escapes, Ryx handles it. If I share, I say shared. If I need raw power, I use unsafe."

memory.ryx
// Region-based allocation: freed at act exit in O(1)
act compute() {
    a := vec[int]()
    b := Map[str, int]()
    c := User { id 1, name "Ryx" }
} // ← a, b, c all freed here — one pointer reset

// Escape analysis: compiler auto-promotes escaping values
act make_user() User {
    u := User { id 1, name "Ryx" }
    retn u   // u escapes → compiler moves to heap
}            // no annotation, no keyword

// Move semantics — single ownership by default
u := User()
v := u       // u is moved to v
// println(u.name) ← compile error: u is moved

// Shared — explicit reference counting
shared node := Node()
a := node    // ref count: 2
b := node    // ref count: 3

// defer — guaranteed resource cleanup
act read_file(path str) Result[str, Error] {
    f := File.open(path)?
    defer f.close()          // runs on scope exit
    content := f.read()?
    retn Ok(content)
}

// unsafe — explicit raw memory
unsafe {
    buf := alloc(1024)
    write_data(buf)
    free(buf)
}

Error Handling

Ryx uses Result[T, E] and Option[T] for explicit, safe error handling. No exceptions.

errors.ryx
// Result[T, E] — success or error
act divide(a float, b float) Result[float, str] {
    if b == 0.0 { retn Err("Division by zero") }
    retn Ok(a / b)
}

// ? operator — early return on error
act process() Result[int, Error] {
    data  := fetch_data()?    // if Err → return Err
    value := parse(data)?     // same
    retn Ok(value)
}

// Pattern matching on Result
match divide(10.0, 3.0) {
    Ok(result)  -> println("Got: {result}"),
    Err(reason) -> println("Failed: {reason}"),
}

// Option[T] — present or absent (no null)
act find_user(id int) Option[User] {
    users := load_db()
    retn users.get(id)
}

// Using Option
match find_user(42) {
    Some(user) -> println("Found: {user.name}"),
    None       -> println("Not found"),
}

// unwrap_or — provide a default
user := find_user(0).unwrap_or(User { id 0, name "Guest" })

// Chain operations
name := find_user(1)
    .map(|u| u.name)
    .unwrap_or("Unknown")

Async & Await

Ryx has zero-cost async with the async/await keywords and the async pipeline operator ~>.

async.ryx
use std.net
use std.json

// Async function
async act fetch_data(url str) Result[str, Error] {
    response := net.get(url).await?
    retn Ok(response.body())
}

// Async pipeline with ~> operator
// Each function receives the output of the previous
async act process_user(id int) Result[User, Error] {
    user := fetch_user(id)
        ~> validate_data()
        ~> enrich_profile()
        ~> cache_result()?

    retn Ok(user)
}

// Await multiple concurrently
async act main() {
    // Parallel fetches
    (a, b, c) := await(
        fetch_data("https://api.ryx.dev/a"),
        fetch_data("https://api.ryx.dev/b"),
        fetch_data("https://api.ryx.dev/c"),
    )

    // Handle results
    match process_user(42).await {
        Ok(user)  -> println("User: {user.name}"),
        Err(e)    -> println("Error: {e}"),
    }
}

Modules

Every Ryx file is a module. Folders are namespaces. Everything is private by default — use pub to export.

modules.ryx
// math.ryx — export public items
pub struct Vector { x float, y float, }
pub act add(a int, b int) int -> a + b
act internal() { }   // private — only usable in math.ryx

// main.ryx — import entire module
use math
sum := math.add(10, 20)

// Import specific items
use math { add, Vector, }
v := Vector { x 1.0, y 2.0, }

// Folder namespace — utils/network.ryx
use utils.network
utils.network.connect("https://ryx.dev")

// Rename on import
use utils.network as net
net.connect("https://ryx.dev")

// Root imports — from project root with ~
use ~.config                    // project/config.ryx
api_key := config.get_api_key()

// mod.ryx — folder entry point
// database/mod.ryx re-exports items:
pub use connection { connect, }
pub use query { run_query, }

// Then from anywhere:
use database
db := database.connect()
database.run_query(db, "SELECT *")
